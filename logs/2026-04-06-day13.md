# Day13 Log - Dual Encoder Serial Monitoring Bring-Up

## Summary
Today I completed the bring-up of dual-wheel encoder serial monitoring on the STM32-based mobile robot platform.

## Work Done

### 1. Single Encoder Feedback Verification
- First verified one motor encoder channel independently
- Confirmed encoder power, ground, and A/B phase wiring were correct
- Successfully read encoder state changes through STM32 GPIO inputs
- Verified serial output on the PC side through the monitor tool

### 2. Dual Encoder Monitoring Bring-Up
- Expanded from single-wheel verification to dual-wheel monitoring
- Connected both left and right motor encoders to STM32
- Confirmed both wheels could be observed independently through serial output
- Verified that encoder feedback from both sides was visible and distinguishable

### 3. Encoder Count Test
- Moved from raw A/B phase state observation to pulse counting
- Implemented basic count accumulation for both wheels
- Verified that wheel motion could be translated into incremental count values
- Confirmed the feedback path from encoder -> STM32 -> serial monitor was working

### 4. Directional Count Logic
- Added basic quadrature direction judgment using A/B phase relationship
- Tested forward/reverse count behavior
- Corrected one wheel's direction sign after observing reversed counting behavior
- Established an initial usable directional count version

## Key Findings
- The serial monitoring chain is now stable and usable for debugging
- Dual-wheel encoder feedback has been successfully brought up
- The system can already support observation of wheel motion, pulse count, and direction
- At higher manual wheel speeds, count behavior becomes unstable, which suggests the current polling-based approach is reaching its limit
- This indicates the next stage should move toward more robust encoder acquisition methods such as interrupt-based reading or timer encoder mode

## Current Project State
- Motor control path: available
- Encoder feedback path: available
- Dual serial monitoring: completed
- Direction-aware counting: initial version completed

## Next Step
- Introduce a more robust encoder reading method
- Improve count stability under higher wheel speed
- Prepare for fixed-period speed estimation
- Later combine feedback with motor control for closed-loop improvement

## Notes
Today’s progress was important because the project has now moved from “the car can move” to “the system can observe wheel feedback in a structured way,” which is a key step toward speed measurement, calibration, and closed-loop control.

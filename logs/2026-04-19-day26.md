# Day26

## Summary
Today I achieved a major milestone in the offline record pipeline of the STM32 mobile robot system.

## What was done
- Investigated the failure of the RAM-only reporting approach
- Confirmed that reconnecting USB could cause the program to restart, which made RAM-based record retention unreliable
- Switched the design direction from temporary RAM storage to Flash-based persistence
- Verified the minimum Flash read/write path
- Built and tested a Flash record clear tool
- Successfully erased the old test record stored in Flash
- Reflashed the run-once program after clearing Flash
- Verified that the robot could execute the motion sequence once and save the real run result into Flash
- Confirmed that after reconnecting to the computer, the robot did **not** start a second run
- Successfully read back the saved run record through UART after reboot / reconnect

## Key result
The system can now:
- run offline once
- save the run result into Flash
- avoid unintended second execution after USB reconnect
- print the last saved record through serial monitoring

## Verified record
- dA = 1063
- dB = 958
- distA = 556 mm
- distB = 501 mm
- avg = 529 mm

## Why this matters
This is the first stable version of:
- offline motion execution
- odometry result persistence
- reboot-safe result retrieval

This means the project has moved beyond temporary monitoring and entered a more system-level stage.

## Current status

# Day15 Log - Encoder EXTI Upgrade and Fixed-Period Speed Feedback

## Summary
Today I upgraded the dual-wheel encoder acquisition path from polling-based sampling to EXTI interrupt-based counting, then further extended the feedback chain to fixed-period speed monitoring and open-loop consistency verification.

## Work Done

### 1. Upgraded Encoder Acquisition to EXTI Interrupt Mode
- Reconfigured encoder A-phase inputs from normal GPIO input to external interrupt mode:
  - `PA7 -> EXTI7`
  - `PC1 -> EXTI1`
- Verified related NVIC interrupt entries were enabled
- Regenerated project code from CubeMX
- Replaced polling-based edge detection in `while(1)` with interrupt-based counting in `HAL_GPIO_EXTI_Callback()`

### 2. Kept B-Phase Direction Judgment
- Retained B-phase pins as normal inputs:
  - `PB6` for A wheel direction judgment
  - `PC0` for B wheel direction judgment
- Preserved the quadrature decoding strategy:
  - trigger count on A-phase rising edge
  - determine sign from B-phase state

### 3. Built Fixed-Period Speed Estimation
- Added fixed-period delta calculation every `100 ms`
- Introduced:
  - `a_count_last`
  - `b_count_last`
  - `a_delta`
  - `b_delta`
- Changed serial output from cumulative count only to:
  - total count
  - delta count per 100 ms
- This established the first usable wheel speed feedback layer

### 4. Added Automatic PWM Ramp Test
- Implemented a forward motor test with automatic PWM stepping
- Used staged PWM progression for observation:
  - `200 -> 300 -> 400 -> 500`
- Printed current PWM together with wheel count and delta data
- This allowed direct observation of left/right wheel speed consistency under equal command input

### 5. Corrected Motion Direction Definition
- Verified that encoder counting direction itself was correct
- Identified that motor motion direction definition was not fully aligned with expected forward motion
- Adjusted B motor forward direction logic in control code without changing encoder sign logic
- Clarified the distinction between:
  - encoder count sign
  - motor drive direction definition

### 6. Observed Open-Loop Wheel Mismatch and Compensation
- Confirmed that under equal PWM, hardware differences caused A wheel speed to remain lower than B wheel by about 10%
- Added simple open-loop compensation by increasing A-side PWM
- Re-ran the ramp test and monitored `dA / dB` and cumulative count consistency

### 7. Verified Improved Consistency
- After compensation, left/right wheel delta values became much closer
- Cumulative count difference remained very small relative to total count
- At total count around `5.8e4`, left/right count gap stayed only around several dozen counts
- This indicates that the current open-loop consistency is already at a good level for the present stage

## Key Findings
- EXTI interrupt mode is a more appropriate architecture than polling for encoder acquisition
- Fixed-period delta feedback is much more useful than raw cumulative count for later control work
- Equal PWM does not guarantee equal wheel speed because of real hardware differences
- Simple open-loop compensation can already significantly improve left/right consistency
- The project has now moved from “encoder can count” to “chassis has usable speed feedback and basic consistency tuning”
